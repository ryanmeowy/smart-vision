`Doubao-embedding-vision` VS `qwen3-vl-embedding`

| 维度             | Doubao-embedding-vision | qwen3-vl-embedding              | 判断                                                 |
| -------------- | ----------------------- | ------------------------------- | -------------------------------------------------- |
| 当前项目匹配度        | 高                       | 很高                              | 两个都能做 text + image 检索，但 qwen3-vl-embedding 更像“知识库检索主模型” |
| 接入速度           | 很高                      | 很高                               | Doubao 更适合快速落地                                     |
| 多模态覆盖          | 文本 / 图片 / 视频            | 文本 / 图片 / 视频，且支持融合向量            | qwen3-vl-embedding 上限更高                            |
| 图文融合能力         | 有统一多模态表达                | 明确支持 `enable_fusion=true` 的融合向量 | qwen3-vl-embedding 更适合后续对话式检索                      |
| 向量维度灵活性        | 中                       | 很高                              | qwen3-vl-embedding 维度调节更灵活                         |
| 长文本/复杂检索扩展性    | 中高                      | 高                               | qwen3-vl-embedding 更适合后续 text-heavy 场景             |
| 作为“首发主模型”的稳妥度  | 高                       | 很高                              | 偏向 qwen3-vl-embedding                              |
| 作为“快速落地模型”的合适度 | 很高                      | 高                               | Doubao 更像快攻方案                                      |
| 后续接本地开源路线的一致性  | 中                       | 高                               | 本地备选 Qwen / GME 路线，qwen3-vl-embedding 更顺        |
| 推荐角色           | Challenger / 快速落地备选     | Main / 主选                       | 先主选 qwen3-vl-embedding，Doubao 做 challenger         |

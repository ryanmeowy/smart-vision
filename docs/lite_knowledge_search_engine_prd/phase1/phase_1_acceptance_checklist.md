# Phase 1 最小验收清单（E6 / P1-22）

> 目标：覆盖上传、任务状态、重试、统一检索与结果展示，验证 text + image 闭环可用。

## 1) 基础链路验收

- [ ] 可提交 `PDF/TXT/MD` 文本任务：`POST /api/v1/ingestion/text-assets/batch-tasks`
- [ ] 文本任务状态可查询：`GET /api/v1/ingestion/text-assets/batch-tasks/{taskId}`
- [ ] 文本任务失败项可重试（单项/全量）
- [ ] 文本 chunk 写入统一索引后，`/api/v1/search/kb` 可返回 `TEXT_CHUNK`
- [ ] 图片入库后，`/api/v1/search/kb` 可返回 `IMAGE_CAPTION`/`IMAGE_OCR_BLOCK`
- [ ] 同一 query 下可同时出现文本结果和图片结果

## 2) 结果协议验收

- [ ] 返回字段包含：`resultType/assetType/snippet/pageNo/score/explain`
- [ ] 文本结果有 `snippet`，且 `pageNo` 正常（有页码时）
- [ ] 图片结果有摘要型 `snippet`
- [ ] `explain.hitSources` 可读（例如 `VECTOR/TITLE/CONTENT/OCR`）

## 3) 最小查询集（12 条）

> 使用前请确认数据集里已有对应文本与图片素材；以下 query 保持 10~20 条范围内，按三类均衡覆盖。

### B. 图片命中为主（4 条）

# PRD 3：Engineering Enhanced 版本（增量强化版）

## 1. 目标定义

这一阶段不是为了再堆功能，而是为了把项目从“强个人项目”再往“高级工程师作品”推进半步。

重点是三件事：

* 可重建
* 可调优
* 可观测

## 2. 核心增量需求

### 2.1 索引与数据版本化

当你未来调整：

* chunk 策略
* embedding 模型
* 文本解析策略
* OCR 策略

系统应能区分不同版本的结果。

要求：

* segment 增加 version 字段
* ingest job 记录 pipeline version
* 索引支持 version 标识

### 2.2 局部重建能力

利用你现有任务框架，把“重试单条图片”扩展成“重建单个 asset / 单批文本资源”的正式能力。你现在接口层已经有任务状态和 item 级重试，这给重建机制留了很好的入口。([GitHub][2])

要求：

* 支持按 asset 重建
* 支持按类型重建（仅文本 / 仅图片）
* 支持失败后继续

### 2.3 检索参数实验化

把关键检索参数配置化：

* chunk size
* overlap
* recall topN
* rerank topK
* text/vector 权重
* OCR 开关策略

目标：

* 让项目具备真正的“搜索调优”味道

### 2.4 更完整的指标面

你当前已经有策略选择和 fallback 指标接入。强化版要把指标从“存在”升级成“能分析问题”。([GitHub][5])

新增建议指标：

* 搜索总耗时 P50 / P95
* 召回链路耗时拆分
* rerank 耗时与命中率
* 文本入库耗时
* OCR 调用耗时
* 重试比例
* 各类命中来源占比

### 2.5 实验报告

至少形成 2 组正式实验对比：

* BM25 only vs hybrid
* hybrid vs hybrid + rerank
* 小 chunk vs 大 chunk
* OCR on vs OCR off

这一步是让你在面试里能讲“我怎么做权衡”，不是只讲“我会接模型”。

## 3. 本阶段不做什么

仍然不建议在这里引入：

* 视频
* 多租户
* SaaS 权限体系
* 在线学习排序
* 复杂前端平台化

## 4. 验收标准

满足以下条件视为 Phase 3 完成：

1. 支持按版本管理 segment / ingest 结果
2. 支持局部重建或重索引
3. 检索参数可配置并能做实验
4. 有一组可复盘的性能与效果指标
5. 有完整实验报告或调优记录

## 5. 产出物

* versioned ingest / index 设计
* 局部重建接口
* 配置化参数清单
* 指标面板或统计报告
* 调优实验文档
